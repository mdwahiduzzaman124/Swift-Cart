#### 1) What is the difference between `null` and `undefined`?
      ANS: Null এবং undefined উভয় একই অর্থ প্রকাশ করে। উভয়ের অর্থ হচ্ছে "কিছুই নয়" অথবা "খালি জায়গা"। কিন্তু এদের ব্যবহার হয় ভিন্নভাবে।

Undefined মূলত ব্যবহার করে JavaScript নিজে থেকে সক্রিয়ভাবে। উদাহরণস্বরূপ Let x; এখানে "x" এর কোন মান দেওয়া হয়নি, যার কারণে JavaScript নিজে থেকে এর মান undefined ধরে নেয়।

Null ব্যবহার হয় developer এর মাধ্যমে। এর অর্থ হচ্ছে "কিছু নয়" অথবা "খালি জায়গায়" কিন্তু এটি মূলত একটি Object. যদি আমরা Null টাইপ চেক করি তাহলে পাব object।

সুতরাং দুটির অর্থ একই হলেও এদের ব্যবহার এবং প্রকারভেদ আলাদা।

#### 2) What is the use of the `map()` function in JavaScript? How is it different from `forEach()`?
      ANS: map() এটি JavaScript এর ES6 এর একটি ফাংশন। এটি যেকোনো Array এর উপর ব্যবহার করা যায়। এটি মূলত কোন একটি Array এর প্রত্যেকটি উপাদানের উপর একটি call back function চালায়। এবং একটি নতুন Array return করে। যদি কোন ডাটা সার্ভারে ডাটা নিয়ে সেটিকে সময়ের সাথে পরিবর্তন এবং আপডেট করার প্রয়োজন পড়ে তাহলে এটি ব্যবহার করাই ভালো এবং সহজ।

forEach() এটি একটি loop এর মতো কাজ করে। কিন্তু এটি কোন কিছু return করে না। যদি আমরা কখনো ডাটা সার্ভার থেকে ডাটা নিয়ে সেটিকে loop করে শুধু প্রদর্শন করতে হয় অথবা কাজ করতে হয় সে ক্ষেত্রে আমরা এটি ব্যবহার করতে পারি।

map() নতুন রিটার্ন করে এবং ডাটা মডিফাই করা সম্ভব। forEach() কোন কিছু রিটার্ন করে না এবং ডাটা নিয়ে যেকোনো কাজ করা যায়।

#### 3) What is the difference between `==` and `===`?
      ANS: (==) And (===) এর মধ্যে পার্থক্য:

(==): শুধুমাত্র মান তুলনা করে। অনেক সময় প্রয়োজনে টাইপ পরিবর্তন করে তুলনা করে যেমন console.log(5 == '5');। Output: True। ব্যবহারের ক্ষেত্রে অনেক সময় নির্ভুল ফলাফল দিতে অক্ষম কারণ চেক স্ট্যাটাস সেক্রিফাইস করে

(===): মান এবং ডেটা টাইপ উভয় তুলনা করে। ডেটা টাইপ চেক করার কারণে এবং পরিবর্তন না করার কারণে এটি বেশি সিকিওর এবং ব্যবহারযোগ্য নির্ভুল এবং নিরাপদ তথ্য প্রদানে সবচেয়ে বেশি উপযোগী। যেমন: console.log(5 == '5');। Output: False।

#### 4) What is the significance of `async`/`await` in fetching API data?
      ANS: JavaScript এ API থেকে ডাটা fetch করার ক্ষেত্রে async and await ব্যবহার করা হয় আরো সহজে বুঝতে এবং দ্রুত ব্যবহার করার জন্য। এটি মূলত fetch promise হ্যান্ডেল করার একটি নতুন পদ্ধতি যা ES6 এরপর থেকে ব্যবহৃত হচ্ছে।

JavaScript এ API থেকে ডাটা লোড করার জন্য পূর্বে ব্যবহার হতো fetch .then .then এমন ধরনের প্রমিস। যা ব্যবহারকারীর পড়া এবং বুঝার ক্ষেত্রে একটু কষ্টদায়ক হওয়ার পাশাপাশি ডিবাগিন/Debug অথবা API related কোন প্রকার সমস্যা তাৎক্ষণিক খুঁজে বের করতে একটু সময় লাগতো যার ফলে। পরবর্তীতে JavaScript একটি নতুন পদ্ধতি নিয়ে আসে যা একে ব্যবহারকারীর পড়া এবং বোঝার ক্ষেত্রে অত্যন্ত সহজ হয় এবং কোন প্রকার API related সমস্যা খুঁজে বের করা এবং তাৎক্ষণিক তার সমাধান করার জন্য সময় কম লাগে এবং বুঝতে সুবিধা হয়। এর অন্যতম সুবিধা হচ্ছে একত্রে অনেকগুলো API ব্যবহার করার সময় এটি আরো সহজে ব্যবহার করা যায় পাশাপাশি কোন সমস্যা হলে তাৎক্ষণিক সেটি কোথায় হচ্ছে খুঁজে বের করার সহজ হয়।

#### 5) Explain the concept of Scope in JavaScript (Global, Function, Block).
      ANS: JavaScript এ ভেরিয়েবল কোথায় তৈরি করা হয়েছে এবং সেটা কোথায় কোথায় ব্যবহার করা যাবে একে বলা হয় Scope।
JavaScript এ মূলত তিন ধরনের স্কোপ আছে:

১. Global scope: যদি কোন ভেরিয়েবল কোন ফাংশন বা ব্লকের বাহিরে ডিক্লেয়ার করা হয় সেটি হচ্ছে গ্লোবলের স্কোপ।
একে আমি যে কোন ফাংশনের ভিতর থেকে চাইলে ব্যবহার করতে পারি।

২. Function scope: যখন কোন ভেরিয়েবল একটি ফাংশনের ভেতরে ডিক্লেয়ার করা হয় তখন সেটি শুধু সে ফাংশনের ভিতরেই কাজ করে এবং ফাংশনের বাহির থেকে ব্যবহার করা সম্ভব হয় না।

৩. Block Scop: JavaScript es6 let এবং const এ দুটি হচ্ছে ব্লক স্কোপ। শুধুমাত্র ওই ব্লকের ভেতরে ব্যাবহার করা যায়।
